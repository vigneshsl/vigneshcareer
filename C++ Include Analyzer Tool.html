🔎 What Is It?

This tool extracts all #include statements from selected .cpp and .h files and saves them in a neat CSV file.

🎯 Why Use It?

  • ✅ Quickly analyze header dependencies
  • ✅ Useful for code cleanup and architecture review
  • ✅ No manual searching needed

✨ Features

  • ✅ Supports .cpp and .h files
  • 📥 Uses a file dialog for easy selection
  • 📄 Outputs results in a timestamped CSV file
  • 📂 Automatically opens the result after completion

👣 How To Use It?

1

Run the Tool

Open the Python script using your editor or run it in terminal.

2

Select Files

When prompted, select one or more .cpp and .h files.

3

Wait and View Results

The tool will analyze each file and extract all #include statements.

Results will be saved to a CSV and opened for you automatically.

📁 Output Format

Output is saved in:

C:\Customization tool\C++ Include Analyzer

The CSV file includes two columns:

  • File Name – Name of the source file
  • Include Statement – The full line with the #include directive
📝 Note: Each run creates a new file with the current date and time in the name.

🧩 Common Problems & Fixes

❌ No files selected
You must select files to extract includes from. Restart the tool and select valid files.
⚠️ File read error
Check that the file isn't locked or encoded strangely (use UTF-8 for best results).

💡 Helpful Tips

📁

Batch Files

You can select multiple files at once to analyze together.

🔍

Open the CSV in Excel to filter or find duplicate headers easily.

🧪

Start with a few files to verify it works as expected before running large batches.
